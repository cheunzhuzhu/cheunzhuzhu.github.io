ENUM和SET的区别：ENUM类型的数据的值，必须是定义时枚举的值的其中之一，即单选，而SET类型的值则可以多选。

二、SQL的约束

约束分类

约束是一种限制，它通过对表的行或列的数据做出限制，来确保表的数据的完整性、唯一性。

比如，规定一个用户的用户名不能为空值没有重复的记录，这就是一种约束

在MySQL中，常见的约束：

约束类型： 主键 默认值 唯一 外键
关键字： PRIMARY KEY DEFAULT UNIQUE FOREIGN KEY

主键（PRIMARY KEY）

在数据空中，如果有两行记录数据完全一样，那么对于MySQL就会认定他们是同一个实体，这与显示生活是存在差异的，这个时候就要用到主键。

主键（PRIMARY KEY）作为数据表中一行数据的唯一标识符，在一张表中通过主键就能确定定位到某一行数据，因此主键十分重要的，它不能有重复记录且不能位空。

还有一种特殊的主键——复合主键。主键不仅可以是表中的一列，也可以由表中的两列或多行来共同标识。

定义主键：CONSTRAINT [主键name] PRIMARY KEY [复合主键name]

默认值约束（DEFAULT）

默认值约束（DEFAULT）规定，当有DEFAULT约束的列，插入数据为空时，将使用默认值。

默认值常用于一些可有可无的字段，比如用户的个性签名，如果用户没有设置，系统给他应该设定一个默认文本，比如空文本或默认参数。

DEFAULT约束只会在使用INSERT语句时体现出来，INSERT语句中，如果被DEFAULT约束的位置没有值，那么这个位置就会被DEFAULT的值填充。

唯一约束（UNIQUE）

唯一约束（UNIQUE）比较简单，它规定一张表中指定的一列的值必须不能有重复值，即这一列每个值都是唯一的。

当INSERT语句插入新的数据和已有数据重复的时候，如果有UNIQUE约束，则INSERT失败。

外键约束（FOREIGN KEY）

外键（FOREIGN KEY）既能确保数据完整性，也能体现表之间的关系。

比如，现在有用户表和文章表，给文章表中添加一个指向用户的id的外键，表示这篇文章所属的用户id，外键将确保这个外键指向的记录是存在的，如果你尝试删除一个用户，而这个用户还有文章存在于数据库中，那么操作将无法完成并报错。因为你删除了该用户过后，他发布的文章就没有所属用户了，而这样的情况是不被允许的。同理，你在创建一篇文章的时候也不能为它指定一个不存在的id。

一个表可以有多个外键，每个外键必须REFERENCES（参考）另外一个表的主键，被外键约束的列，取值必须在它参考的列中有对应值。

CONSTRAINT [外键name] FOREIGN KEY(设为外键的字段) REFERNCES [参考某表中的（某列)]

在INSERT时，如果被外键约束的值没有在参考列中有对应，则INSERT失败。

非空约束（NOT NULL）

非空约束（NOT NULL）被非空约束的列，在插入值中必须非空。

在MySQL中违反非空约束，会报错。

三、SELECT 语句详解

基本的SELECT语句

SELECT 要查询的列名 FROM 表名字 WHERE 限制条件；

如果要查询表的所有内容，则把 要查询的列名 用一个星号（*）表示，代表要查询表中所有的列。例如要查询lianxi表的name和phone：

SELECT name,phone FROM lianxi;

数学符号条件

SELECT语句尝尝会有WHERE限制条件，用于达到更加精确的查询。WHERE限制条件可以有数学符号(=,<,>,>=,<=)，假设要查询id为1的name和phone信息：

SELECT name,phone FROM lianxi WHERE id=1;

例如要查询name为Rose的信息：

SELECT * FROM lianxi WHERE name='Rose';

“AND” 与 “OR”

在此之前现在lianxi表中添加新的字段 age

然后再更新数据：

再利用 INSERT INTO 添加新的数据

WHERE后面可以有不止一条限制而根据他们之间的逻辑关系，可以用[条件一 OR 条件二] 和 [条件一 AND 条件二]连接:

筛选出 age 小于25，或age大于26的人员信息：

SELECT * FROM lianxi WHERE age<25 OR age>26;

筛选出 age 大于23，且 age 小于 27的人员信息：

SELECT * FROM lianxi WHERE age>23 AND age<27;

刚才的限制条件 age>23 AND age<27,如果需要包含25和30这两个数字的话，可以替换为 age BETWEEN 23 AND 27:

IN 和 NOT IN

关键词 IN 和 NOT IN的作用是用于筛选“在”或者“不在”某个范围内的结果，例如查询age是25 或 27 的人员信息:

SELECT * FROM lianxi WHERE age IN (25,27);

或者查询出age不是25也不是23的人员信息:

SELECT * FROM lianxi WHERE age NOT IN (25,23);

通配符

关键字 LIKE 可以用于实现模糊查询，常见于搜索功能中。

和 LIKE 联用的通常还有通配符，代表未知字符。SQL中的通配符是**_和%**。其中 _ 代表一个未定义字符，%代表不定个未指定字符

比如，只记得电话号码前七位数是1101101，而后两位忘记了，则可以用两个 _ 通配符代表:

SELECT * FROM lianxi WHERE phone LIKE '1101101__';

例如只记得名字首字母，又不知道名字的长度，则可以用%通配符代表不定个字符:

SELECT * FROM lianxi WHERE name LIKE 'R%';

对结果排序

为了使查询结果看起来更顺眼，我们可以对结果按照某一列来排序，这就用到了 ORDER BY 排序关键词，默认情况下 ORDER BY的结果是升序排列，而使用关键词ASC和DESC可以指定升序或者降序排列

例如，我们按照age降序排列：

SELECT * FROM lianxi ORDER BY age DESC;

SQL内置函数的计算

SQL允许对表中的数据进行计算。对此SQL有5个内置函数，这些函数都对SELECT的结果做操作：

函数名： COUNT SUM AVG MAX MIN
作用： 计数 求和 求平均值 最大值 最小值

其中COUNT函数可以用于任何数据类型（因为他只是计数），而SUM、AVG函数只能对数字类型数据类型做计算，MAX和MIN可以用于数值、字符串或是日期时间数据类型

例如，计算出age的最大值和最小值：

SELECT MAX(age) AS max_age,MIN(age) AS min_age FROM lianxi;

四、数据库及表的修改和删除

删除数据库

在此之前需要先新建一个数据库帮助实验CREATE DATABASE test_1;，使用命令 SHOW DATABASS；可以看到刚才新建的数据库。

然后再删除刚才新建的test_1数据库使用命令 DROP DATABASE [库名];

目前Mysql没有提供修改数据库名称的方法，因为这曾导致一系列的安全问题。

再老版本中RENAME DATABASE可以修改数据库名称，这条命令再MySQL5.1.7中被加入，但官方发现这命令的危险，于是再MySQL5.1.23中移除

对一张表的修改

重命名一张表

重命名一张表的语句有多种,以下三种效果一致

在此之前先新建一张表 CREATE TABLE table_1(id int(10),name char(20));

使用命令将table_1的名字修改为table_2 RENAME TABLE table_1 TO table_2;

删除表

删除表的语句和删除数据库的语句大致相同 DROP TABLE 表名;

对一列的修改（即对表结构的修改）

新增一列

再表中增加一列的语句格式为:

在原有的表中加入height一列并指定DEFAULT约束为170：

ALTER TABLE lianxi ADD height INT(4) DEFAULT 170;

新增列默认放置在这张表的最右边。如果要吧增加的列插入在指定位置，则需要再语句的最后使用AFTER关键词

例如我们要新增一列weight放置在age的后面：

ALTER TABLE lianxi ADD weight INT(4) DEFAULT 120 AFTER age;

把新增列放在第一列的位置可以使用关键词FIRST关键词

ALTER TABLE lianxi ADD test INT(10) DEFAULT 11 FIRST;

删除一列

删除表中的一列和刚才使用的新增一列的语句格式十分相似，只是把关键词ADD改成了DROP，语句后面不需要有数据类型、约束或位置信息。具体格式：

删除刚才新增的test：

ALTER TABLE lianxi DROP test；

重命名一列

这条语句其实不止可以用于重命名一列，准确的来说，它是对一个列做出修改(CHANGE):

ALTER TABLE 表名 CHANGE 原列名 新列名 数据类型 约束;

注意：这条重命名语句后面的“数据类型”不能省略，否则重命名失败。

当原列名和新列名相同的时候，指定新的数据类型或约束，就可以用于修改数据类型或约束。需要注意的是，修改数据类型可能会导致数据丢失，所以要谨慎使用。

我们用这条语句将“height”一列重命名为“shengao”，效果如下：

ALTER TABLE lianxi CHANGE height shengao INT(4) DEFAULT 170;

改变数据类型

要修改一列的数据类型，除了使用刚才的CHANGE语句外，还可以用这样的MODIFY语句：

ALTER TABLE 表名 MODIFY 列名 新数据类型；

修改数据类型时需要小心使用，因为这可能会导致数据丢失。

想要修改表中的某一个或几个数据，这就需要我们用到 UPDATE 这条命令：

UPDATE 表名 SET 列1=值1,列2=值2 WHERE 条件;

删除表中的一行数据，也必须加上WHERE条件，否则整列的数据都会被删除。

DELETE FROM 表名 WHERE 条件;
